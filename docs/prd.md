# 需求文档

## 1. 应用概述

### 1.1 应用名称
4+1教案迭代助手

### 1.2 应用描述
面向幼儿园教师的网页应用，帮助教师按照「4+1」方法高效打磨教案，通过五个步骤模块（搭骨架、填血肉、挑毛病、定终稿、上海味道校准）完成从初稿到终稿的迭代优化。

## 2. 用户与使用场景

### 2.1 目标用户
幼儿园教师

### 2.2 核心使用场景
教师需要编写或优化教案时，通过本应用按照「4+1」方法逐步完成教案的生成、优化、问题诊断、整合修改和风格校准，最终输出符合《3-6岁儿童学习与发展指南》要求的高质量教案。

## 3. 页面结构与功能说明

### 3.1 页面结构

```
首页（我的教案·历史记录）
├── 新建教案入口
│   └── 第一步：搭骨架·出初稿
│       └── 第二步：填血肉·做丰满
│           └── 第三步：挑毛病·找真问题
│               └── 第四步：定终稿·做整合
│                   └── 第五步：上海味道校准
│                       └── 完成页面
└── 历史教案详情页
    └── 版本记录查看
```

### 3.2 首页（我的教案·历史记录）

#### 3.2.1 新手引导
首次使用时显示引导说明，内容包括：
  - 「4+1」方法是什么
  - 五个步骤分别做什么

#### 3.2.2 新建教案入口
提供「新建教案」按钮，点击进入第一步

#### 3.2.3 历史教案列表
  - 按时间倒序排列
  - 每项显示：教案标题、年龄段、领域、最后修改时间、当前状态标签（初稿/优化中/已终稿/已校准）
  - 支持搜索功能
  - 支持删除功能
  - 点击教案项进入历史教案详情页

### 3.3 第一步：搭骨架·出初稿

#### 3.3.1 进度指示
顶部显示细线进度条，当前步骤用砖红色高亮

#### 3.3.2 表单填写
  - 年龄段：下拉选择（托班/小班/中班/大班）+ 细分描述输入框
  - 活动领域：多选（科学/语言/艺术/健康/社会）
  - 活动时长：数值输入（分钟）
  - 活动主题：文本输入
  - 核心目标2-3个：认知目标、能力目标、情感目标，分别提供文本输入框

#### 3.3.3 生成初稿
  - 点击「生成初稿」按钮
  - AI根据表单信息生成完整教案框架（包含活动目标、活动准备、活动过程、活动延伸、观察要点），符合《3-6岁儿童学习与发展指南》，体现儿童本位、游戏为基本活动
  - 生成后展示教案内容，支持用户预览和微调
  - 提供「确认，进入下一步」按钮

### 3.4 第二步：填血肉·做丰满

#### 3.4.1 进度指示
顶部进度条更新至第二步

#### 3.4.2 教案展示
展示当前教案内容

#### 3.4.3 优化方向选择
用户勾选希望优化的方向（多选）：
  1. 教师语言幼儿化
  2. 增加师幼互动示例
  3. 强化游戏仪式感
  4. 分层指导策略
  5. 易错点应对

#### 3.4.4 自定义优化要求
提供文本框，用户可补充自定义优化要求

#### 3.4.5 开始优化
  - 点击「开始优化」按钮
  - AI在保持原有结构和时长不变的基础上，将教案变得更具体、更落地、更有画面感
  - 优化后支持左右对比查看修改前后的差异
  - 提供「确认，进入下一步」按钮
  - 提供「返回上一步」按钮

### 3.5 第三步：挑毛病·找真问题

#### 3.5.1 进度指示
顶部进度条更新至第三步

#### 3.5.2 教案展示
展示当前教案内容

#### 3.5.3 维度选择
用户选择AI从哪几个维度找茬（默认全选）：
  1. 目标匹配度：环节是否支撑目标
  2. 年龄适宜性：是否匹配该年龄段
  3. 儿童参与度：孩子是否主动探究
  4. 安全与常规：有无安全隐患
  5. 差异性支持：是否关注不同水平孩子
  6. 效果可评估：有没有可操作的评估方法

#### 3.5.4 问题清单生成
  - AI从每个角度生成问题清单
  - 每条问题包含：问题描述、原因分析、改进建议

#### 3.5.5 问题标记
  - 用户对每条问题标记「可接受」「需修改」或「驳回」（用三个不同颜色的按钮）
  - 支持用户手动添加自己发现的问题
  - 提供「确认，进入下一步」按钮
  - 提供「返回上一步」按钮

### 3.6 第四步：定终稿·做整合

#### 3.6.1 进度指示
顶部进度条更新至第四步

#### 3.6.2 问题汇总
系统自动汇总上一步中标记为「需修改」的所有问题

#### 3.6.3 补充修改要求
提供文本框，用户可额外补充修改要求

#### 3.6.4 生成终稿
  - 点击「生成终稿」按钮
  - AI针对所有问题做修改，输出完整干净的终版教案
  - 用户预览确认
  - 提供「确认，进入下一步」按钮
  - 提供「返回上一步」按钮

### 3.7 第五步：上海味道校准

#### 3.7.1 进度指示
顶部进度条更新至第五步

#### 3.7.2 校准维度选择
提供三个校准维度，用开关按钮独立勾选：
  1. 儿童主体性校准：把教师指令式语言改为引导式、开放式的语言
  2. 生成空间校准：增加弹性预设，让教案不是死板的剧本
  3. 收放平衡校准：明确教师介入边界，留白等待时间

#### 3.7.3 一键校准
  - 点击「一键校准」按钮
  - AI做润色（不是加料），输出「润物细无声」的上海风味教案
  - 支持校准前后对比展示
  - 提供「完成」按钮
  - 提供「返回上一步」按钮

### 3.8 完成页面

#### 3.8.1 教案展示
展示完整教案内容和迭代历程

#### 3.8.2 操作功能
  - 一键复制教案全文
  - 分享功能：支持将完成教案分享为图片或文字
  - 返回首页

### 3.9 历史教案详情页

#### 3.9.1 教案内容展示
展示完整教案内容

#### 3.9.2 版本记录
展示每次迭代的版本记录，用户可查看不同版本

#### 3.9.3 操作功能
  - 一键复制教案全文
  - 分享功能：支持将教案分享为图片或文字
  - 返回首页

## 4. 业务规则与逻辑

### 4.1 步骤流程规则
  - 用户必须按顺序依次完成第一步到第五步
  - 每一步完成后点击「确认，进入下一步」按钮才能进入下一步
  - 每一步都可以返回上一步修改
  - 完成第五步后，展示完整教案和迭代历程

### 4.2 AI生成规则
  - 第一步生成的教案框架必须符合《3-6岁儿童学习与发展指南》
  - 第二步优化时保持原有结构和时长不变
  - 第五步校准时只做润色不加料

### 4.3 数据存储规则
  - 使用localStorage存储历史记录
  - 每个教案保存所有迭代版本
  - 记录教案的当前状态（初稿/优化中/已终稿/已校准）

### 4.4 AI调用规则
  - 通过调用大模型API实现AI功能
  - 无需后端登录

## 5. 设计风格规范

### 5.1 色彩规范
  - 主色调：温暖米白(#F7F3EB)、砖红色(#8B4513)、暖灰色(#A0937D)
  - 风格：极简MUJI风，温暖安静有质感

### 5.2 字体与排版
  - 字体：圆润亲和
  - 大段文字行间距1.8以上
  - 排版：大量留白，不拥挤，每屏信息量适中

### 5.3 组件样式
  - 卡片设计：圆角卡片，浅米色背景，砖红色细线边框
  - 进度指示：顶部细线进度条，当前步骤用砖红色高亮
  - 教案展示区：使用类纸张微肌理背景

## 6. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户未填写完整表单信息 | 提示用户补全必填项 |
| AI生成失败 | 提示用户重试，保留已填写内容 |
| 用户在第三步未标记任何问题为「需修改」 | 第四步提示用户确认是否直接生成终稿 |
| 用户在第五步未勾选任何校准维度 | 提示用户至少选择一个维度或跳过此步骤 |
| 历史记录为空 | 首页显示空状态提示，引导用户新建教案 |
| 搜索无结果 | 显示「未找到相关教案」提示 |
| 删除教案 | 弹出二次确认提示 |
| 分享功能调用失败 | 提示用户重试或使用复制功能 |

## 7. 验收标准

1. 用户进入首页，点击「新建教案」按钮
2. 在第一步填写完整表单信息（年龄段、活动领域、活动时长、活动主题、核心目标），点击「生成初稿」，AI生成教案框架
3. 在第二步勾选优化方向，点击「开始优化」，AI优化教案内容
4. 在第三步选择维度，AI生成问题清单，用户标记至少一个问题为「需修改」
5. 在第四步系统汇总问题，点击「生成终稿」，AI输出终版教案
6. 在第五步勾选至少一个校准维度，点击「一键校准」，AI输出校准后教案
7. 在完成页面点击「一键复制教案全文」，成功复制教案内容
8. 返回首页，在历史教案列表中看到刚创建的教案记录
